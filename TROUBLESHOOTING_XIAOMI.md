# 小米手机兼容性修复指南

## 问题描述

小米 11（Android 版本 2.0.6.0.UKBCNXM）无法运行软件，但能安装。

## 已实施的代码优化

### 1. SDK 版本调整

- **原版本**: compileSdk = 36, targetSdk = 36
- **新版本**: compileSdk = 34, targetSdk = 34
- **原因**: Android 36 过于激进，降低版本提高兼容性

### 2. R8 编译优化

```properties
android.enableR8.fullMode=false
```

- 禁用 R8 full mode 以减少运行时错误

### 3. Application 初始化

- 新增 `VmqApplication` 类
- 在应用启动时预创建通知渠道
- 避免服务启动时的资源竞争

### 4. 服务启动优化

- MainActivity 延迟 500ms 启动前台服务
- 添加异常捕获和降级方案
- 适配不同 Android 版本的服务启动方式

### 5. 前台服务增强

- 添加 `onStartCommand` 返回 `START_STICKY`
- 服务被杀死后自动重启
- 多层异常处理确保服务可用

## 用户手动修复步骤

### 方案一：关闭 MIUI 优化（推荐）

1. 打开「设置」
2. 进入「更多设置」
3. 选择「开发者选项」
4. 找到「启用 MIUI 优化」
5. 关闭该选项
6. **重启手机**
7. 重新打开应用

### 方案二：完整权限配置

1. **通知权限**
    - 设置 → 应用设置 → 应用管理 → V 免签监控端
    - 权限管理 → 通知读取 → 允许

2. **自启动权限**
    - 设置 → 应用设置 → 应用管理 → V 免签监控端
    - 权限管理 → 自启动 → 允许

3. **后台弹出界面**
    - 设置 → 隐私保护 → 特殊权限设置 → 后台弹出界面
    - 找到应用并开启

4. **电池白名单**
    - 设置 → 省电与电池 → 右上角齿轮 → 应用智能省电
    - 找到应用 → 选择「无限制」

5. **锁屏清理白名单**
    - 手机管家 → 加速清理 → 右上角齿轮 → 忽略的名单
    - 添加应用到白名单

### 方案三：开发者选项调整

1. 开启开发者选项
    - 设置 → 我的设备 → 全部参数
    - 连续点击「MIUI 版本」7 次

2. 调整动画缩放
    - 设置 → 更多设置 → 开发者选项
    - 窗口动画缩放：0.5x 或关闭
    - 过渡动画缩放：0.5x 或关闭
    - 动画程序时长缩放：0.5x 或关闭

3. 关闭内存优化（可选）
    - 开发者选项中查找「内存优化」
    - 关闭该选项

### 方案四：ADB 命令修复（高级）

如果以上方法都无效，可尝试使用 ADB 命令：

```bash
# 连接手机
adb devices

# 禁用 MIUI 优化
adb shell settings put global miui_optimization 0

# 重启应用
adb shell am force-stop com.shinian.pay
adb shell am start -n com.shinian.pay/.ui.MainActivity
```

### 方案五：降级安装

如果最新版仍然无法运行：

1. 卸载当前版本
2. 下载历史版本（如 v2.x）
3. 安装旧版本测试

## 常见问题 FAQ

### Q1: 为什么能安装但不能运行？

A: 这通常是因为：

- SDK 版本不兼容
- MIUI 系统优化导致
- 权限被系统限制
- 服务启动被阻止

### Q2: 关闭 MIUI 优化有什么影响？

A:

- ✅ 应用兼容性更好
- ✅ 系统限制更少
- ⚠️ 可能略微增加耗电
- ⚠️ 部分 MIUI 特性不可用

### Q3: 需要 Root 权限吗？

A: 不需要！本应用完全无需 Root 即可运行。

### Q4: 为什么其他品牌手机没问题？

A:

- MIUI 对后台管理非常严格
- 各品牌系统优化策略不同
- 已针对主流品牌做适配

### Q5: 云手机能用吗？

A:

- 大部分云手机可以正常使用
- 如遇权限问题请联系云手机客服
- 建议使用真机以获得最佳体验
